# DESIGN — Design Library

A reference library of designs and overall visual languages the agent can draw on when building or restyling features. Each entry captures a look: color scheme, border radius, button/card styling, typography, spacing, and the feel it conveys — so future work stays visually consistent and the agent has a palette of proven styles to reuse.

## What belongs here

- **Style snapshots** — a self-contained `.md` describing one coherent visual language (e.g. `neon.md`, `soft.md`).
- **Component references** — how buttons, cards, inputs, and tabs should look under a given style.
- **Moodboards / canvases** — optional `.canvas` visualizing a style's pieces.
- Generated by the `$DESIGN` skill, or added manually.

## Adding an entry

1. Create `<stylename>.md` (or a subfolder) under `DESIGN/`.
2. Document the look per the `$DESIGN` skill's template (colors, radius, buttons, cards, etc.).
3. Link it back to [[main.md]] and the relevant `PLAN/` entry so the graph reflects the relationship.
